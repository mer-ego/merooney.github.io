A blogging alter ego is born... Mr. Mundane

24 February 2015

Why am I going through the trouble of establishing a personal website? I asked myself this question many times over as I perused listings of today's most popular static site generators. Despite the likelihood my current enthusiasm will shrivel, and the rate of updates will become a dribble, I just couldn't shake the compulsion. Upon closer examination, I perceive the following as my motivating reasons:

  • My trade involves programming, and making a professional showcase online is expected of practitioners.
  • A webpage is a convenient, widely accessible canvas upon which to experiment with current design trends.
  • I like writing. By making my writing public, my vanity can be sated.
  • The act of maintaining and updating this site will keep me informed with trending web technologies.
  • Guilt over not posting enough content of value will steer me towards writing technical articles.
  • I need to grow my digital presence free of any closed platforms.
  • By writing such a weighty manifesto in my first post, I will hereafter be shamed into strict adherence.

As of this post, I am generating this site statically using MetalSmith. Since javascript will continue to be the runtime language of the browser, I figure I ought to achieve some level of proficiency with it. It made a lot of sense to use javascript for the site backend as well as the frontend. Truly a novel idea!

I am using the GitHub pages service for hosting because I can't be bothered to serve up my own HTML. Who has time to run their own HTTP servers and keep them patched these days?! I can't be expected to have to do my own "DevOps" these days. If I need to make this website more dynamic I will just shove more javascript into your browser tab, so look forward to that. I am writing the markdown for this post before I have even deployed this site, so I don't know what my frontend styling will look like. I don't want to incur too much technical debt, so I will suffer through writing my own CSS rather than use a massive framework. Full disclosure: I tried using Bootstrap, but since I had never written a CSS theme before, I produced a fragile abomination. I now realize Bootstrap and similar CSS "solutions" are overkill for a personal website. My sympathies go out to all those beginners who are tricked into believing they need several kB of pre-written CSS before they write a single line of their own.

If you are as excited as I am about watching how this blog develops a unique voice, I will be implementing an RSS/(atom?) feed even though I myself have never subscribed to an RSS feed. Let's subscribe together.

The source code for this site can be viewed on my GitHub profile.
